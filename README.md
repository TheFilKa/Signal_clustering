## Краткое описание метода

Сигналы сначала переводятся из ADC-формата в положительные импульсы, после чего для каждого события вычитается индивидуальный baseline. Основной физический признак — PSD:

```text
PSD = (long_gate - short_gate) / long_gate
```

Окна считаются относительно максимума импульса:

```text
pre_shift = -15
short_len = 30
long_len = 120
```

Во время EDA стало видно, что две группы сигналов образуют не прямую границу, а две изогнутые полосы на плоскости `log_area`–`PSD`. Поэтому финальная модель не использует одну глобальную границу. Вместо этого:

1. события делятся на 36 бинов по `log_area`;
2. в каждом бине обучается `GaussianMixture(n_components=2)` только по PSD;
3. верхняя PSD-полоса отправляется как `cluster=0`, нижняя — как `cluster=1`;
4. большой третий кластер не используется, потому что он ухудшал score;
5. дополнительно 350 самых неуверенных объектов из пограничной зоны переводятся из `0` в `1`.

## Проверенные гипотезы

| Подход | Public score | Вывод |
|---|---:|---|
| Базовый GMM с 3 кластерами | 0.44622 | Третий кластер как глобальные выбросы работает плохо. |
| Локальный PSD + 12% low-confidence в `cluster=2` | 0.75109 | Локальная PSD-граница сильно лучше глобального GMM. |
| Локальный PSD без `cluster=2` | 0.83615 | Большой третий кластер оказался вредным. |
| Локальный PSD + 350 border flips | **0.84092** | Лучший результат: небольшая коррекция пограничной зоны. |

## Итог

Финальное решение получилось физически интерпретируемым: оно разделяет две PSD-полосы, которые соответствуют разной форме хвоста импульса. Главный практический вывод: для этих данных важнее аккуратно разделить два основных типа событий, чем выделять большой отдельный класс аномалий.
